# 🎵 Relatório de Check-in Profundo — FREQsys

> **Data:** 25/02/2026 · **Escopo:** Inventário completo, diagnóstico de bugs/fragilidades, avaliação dos MCPs (shadcn/ui + Context7), plano de melhorias em fases, top 3 patches.

---

## 1. Resumo Executivo

O FREQsys é uma aplicação SPA robusta para gestão de produção musical, com **15+ telas/fluxos**, ~**16.000 linhas de JS** e ~**5.500 linhas de CSS**. O projeto possui uma identidade visual forte (4 temas, sidebar animada, landing page com canvas), mas **sofre de problemas estruturais**:

1. **Zero acessibilidade** — nenhum `role`, `aria-*`, `tabindex` ou focus trap em nenhum modal/dialog
2. **Duplicação de código** — `showLoading`/`hideLoading` definido 2× com IDs diferentes
3. **Toast primitivo** — 3 linhas sem tipo visual, sem aria-live, sem possibilidade de ação
4. **Forms sem validação HTML5** — 50+ `.value = ...` set/get diretos, nenhum `required`, `pattern`, ou `<form>` com `submit`
5. **Nenhum sistema de componentes** — todo HTML é inline no `index.html` (5116 linhas)

**shadcn/ui pode ajudar significativamente**, mas requer adoção parcial e incremental — não é viável migrar tudo de uma vez, pois shadcn/ui é projetado para React/Next.js e o FREQsys é vanilla JS puro. **A estratégia correta é extrair os PADRÕES de design do shadcn/ui como inspiração e implementar componentes vanilla equivalentes.**

## 4. Avaliação dos MCPs

### 4.1 shadcn/ui — Estratégia Recomendada

> [!IMPORTANT]
> **shadcn/ui é projetado para React/Next.js com Tailwind CSS.** O FREQsys é **vanilla JS/HTML/CSS puro**. Não é possível simplesmente `npx shadcn add button` e usar. A migração completa exigiria reescrever o projeto inteiro em React — **não recomendado**.

**Estratégia recomendada: *Design Token Extraction + Vanilla Implementation Inspired by shadcn/ui***

Usar shadcn/ui como **referência de design** (tokens CSS, padrões de API, naming conventions) e implementar componentes vanilla que seguem a mesma filosofia. Isso dá ao FREQsys:
- Consistência visual do shadcn sem a dependência de React
- Padrões de acessibilidade do Radix UI (base do shadcn) como blueprint
- Possibilidade de migração futura para React, já com tokens compatíveis

## 6. Top 3 Patches Detalhados

### Patch 1: Unificar `showLoading`/`hideLoading`

**Diagnóstico:** Existem duas implementações — uma local (L898, ID `fb-loading`) e uma `window.*` (L8747, ID `global-loader`). A segunda sobrescreve a primeira em runtime, mas `hideLoading` local (L909) remove `fb-loading`, enquanto `window.hideLoading` (L8805) remove `global-loader`. Se a local `showLoading` for chamada por código que roda antes da definição global, o overlay fica preso.

**Patch sugerido:**

```diff
 // Remover a implementação LOCAL (linhas 897-909 de firebase-init.js):
-// ─── LOADING ──────────────────────────────────────────────────────────────────
-function showLoading(msg = 'Conectando...') {
-  let el = document.getElementById('fb-loading');
-  if (!el) {
-    el = document.createElement('div');
-    el.id = 'fb-loading';
-    el.style.cssText = '...';
-    el.innerHTML = `...`;
-    document.head.insertAdjacentHTML('beforeend', '<style>@keyframes spin{to{transform:rotate(360deg)}}</style>');
-    document.body.appendChild(el);
-  } else { document.getElementById('fb-loading-msg').textContent = msg; }
-}
-function hideLoading() { document.getElementById('fb-loading')?.remove(); }
+// LOADING — usar apenas window.showLoading/hideLoading (definido mais abaixo, L8747+)
+// Funções locais removidas para evitar duplicação de IDs (fb-loading vs global-loader).
```

**Adicionalmente**, mover a definição `window.showLoading` / `window.hideLoading` (L8747-8808) para logo após as imports, ANTES do primeiro uso.

**Como testar:**
1. Abrir o app → tela de loading deve aparecer com animação FREQsys
2. Login → loading deve desaparecer após carregar teams
3. Console: `showLoading('teste')` → overlay aparece → `hideLoading()` → desaparece
4. Verificar que `document.getElementById('fb-loading')` retorna `null` (ID antigo não existe mais)

**Rollback:** Restaurar linhas 897-909 do backup.

### Patch 2: Adicionar acessibilidade ao container de Toasts

**Diagnóstico:** A função `toast()` (L1729) cria divs dentro de `#toasts`, mas este container provavelmente não tem `aria-live`. Segundo WAI-ARIA APG (consultado via Context7), notificações dinâmicas devem usar `aria-live="polite"` ou `role="status"` para que leitores de tela anunciem o conteúdo.

**Patch sugerido — `index.html`:**

Localizar o elemento `<div id="toasts">` e adicionar atributos:

```diff
-<div id="toasts"></div>
+<div id="toasts" role="status" aria-live="polite" aria-atomic="false"></div>
```

**Patch sugerido — `firebase-init.js` (L1729-1732):**

```diff
-function toast(msg, type = 'success') {
-  const el = document.createElement('div'); el.className = `toast ${type}`; el.textContent = msg;
-  document.getElementById('toasts').appendChild(el); setTimeout(() => el.remove(), 3400);
-}
+function toast(msg, type = 'success') {
+  const icons = { success: '✅', error: '❌', info: 'ℹ️', warning: '⚠️' };
+  const el = document.createElement('div');
+  el.className = `toast toast-${type}`;
+  el.setAttribute('role', 'alert');
+  el.innerHTML = `
+    <span class="toast-icon">${icons[type] || icons.success}</span>
+    <span class="toast-msg">${escHtml(msg)}</span>
+    <button class="toast-close" aria-label="Fechar" onclick="this.parentElement.remove()">✕</button>
+  `;
+  // Animação de entrada
+  el.style.animation = 'toastSlideIn 0.3s ease-out';
+  document.getElementById('toasts').appendChild(el);
+  // Auto-remove com animação de saída
+  setTimeout(() => {
+    el.style.animation = 'toastSlideOut 0.3s ease-in forwards';
+    setTimeout(() => el.remove(), 300);
+  }, 3400);
+}
```

@keyframes toastSlideOut { from { transform: translateX(0); opacity: 1; } to { transform: translateX(100%); opacity: 0; } }
.toast { display: flex; align-items: center; gap: 8px; padding: 10px 14px; border-radius: 8px; font-size: 13px; }

### Patch 3: Substituir `document.execCommand('copy')` por Clipboard API

**Diagnóstico:** `document.execCommand('copy')` está deprecated em todos os browsers modernos. O Clipboard API (`navigator.clipboard.writeText()`) é o substituto recomendado, com suporte em 95%+ dos browsers.

**Patch sugerido — `firebase-init.js` (L2952 e L2671):**

```diff
-// L2952 — letra copy
-const ta = document.createElement('textarea'); ta.value = text; document.body.appendChild(ta); ta.select(); document.execCommand('copy'); document.body.removeChild(ta); toast('Texto copiado!');
+// L2952 — letra copy
+navigator.clipboard.writeText(text).then(() => toast('Texto copiado!')).catch(() => { /* fallback */ const ta = document.createElement('textarea'); ta.value = text; document.body.appendChild(ta); ta.select(); try { document.execCommand('copy'); } catch(e) {} document.body.removeChild(ta); toast('Texto copiado!'); });

-// L2671 — similar pattern
-ta.value = text; document.body.appendChild(ta); ta.select();
+navigator.clipboard.writeText(text).then(() => toast('Copiado!')).catch(() => { const ta = document.createElement('textarea'); ta.value = text; document.body.appendChild(ta); ta.select(); try { document.execCommand('copy'); } catch(e) {} document.body.removeChild(ta); });
```

**Como testar:**
1. Abrir qualquer modal de Letra → clicar "Copiar" → colar em outro lugar → texto correto
2. Copiar código de convite → colar → texto correto
3. Verificar console: sem warnings de `execCommand` deprecated

**Rollback:** Reverter para `execCommand('copy')`.

## 8. Observações Finais

> [!WARNING]
> **Não migrar para shadcn/ui diretamente.** O FREQsys é vanilla JS e migrar para React seria uma reescrita completa. Use shadcn/ui como **referência de design** para implementar componentes vanilla equivalentes.

> [!NOTE]
> **Context7 foi consultado** para validar padrões WAI-ARIA APG de dialog/modal. Os padrões exigem: `role="dialog"`, `aria-modal="true"`, `aria-labelledby`, focus trap, ESC close, e focus return — **nenhum existe atualmente no FREQsys.**

> [!TIP]
> **Abordagem incremental é segura:** Cada patch pode ser aplicado separadamente e revertido sem impacto nos demais. Comece pela Fase 1 (quick wins) e valide antes de avançar.
